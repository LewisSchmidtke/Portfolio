Project Overview

â€” Context

This project is currently on-hold as I am working officially for BMW Motorsport.

This project explores the processing and analysis of various motorsport data sources like the openF1 API endpoints and WEC data by alkamelsystems. It aims to visualize the data relations to analyze data from any session, driver, and lap to gain insights into performance, lap times, and on-track events.

Race cars produce an incredible amount of data. Telemetry sensors, system management, GPS data and much more are all constantly broadcasted throughout any session. Analyzing this data and extracting key information requires both engineering expertise and analytical insights to derive meaningful strategies. This project leverages data science techniques to extract meaningful patterns from motorsport data, enabling quick and actionable insights for improved on-track performance.

With my engineering background, I was able to create meaningful visualizations that are relevant for improving performance and monitoring component functionality and health. The plan was to create an interactive dashboard, however like mentioned above is currently on hold.

Methodology

â€” Approach

For this project the data was either directly acquired through openF1 API endpoints or manually downloaded in the case of the WEC Data. The analysis was conducted using Python with Pandas for data manipulation and Seaborn & Matplotlib for visualization purposes. Key calculations were either done driver-specific, car-specific (WEC), team-specific or holistically, to provide a range of performance potentials.
